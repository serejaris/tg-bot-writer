Шпаргалка: цикл работы с агентом

Короткая схема для следующих задач.

Шаг Что просить у агента Что считать доказательством
1. Цель Создать parent issue с понятным результатом. Issue открыт, критерии готовности понятны.
2. Подзадачи Разбить на sub-issues через GitHub Relationships. Связи видны в GitHub, не только в тексте.
3. Контракты Описать входы, выходы, ошибки, данные. По контрактам можно писать тесты.
4. Красные тесты Написать тесты и показать, что они падают. Падение связано с отсутствием функции, а не с опечаткой.
5. Код Сделать минимальную реализацию до зелёных тестов. Тесты проходят локально и на сервере.
6. E2E Проверить реальный пользовательский сценарий. Есть факт: бот ответил, сохранил, опубликовал.
7. Закрытие Коммит, push, комментарий в issue, закрытие. Коммит виден в GitHub issue.

Главное правило: если E2E нашёл баг, это не «мелочь после готовности», а новая sub-issue.
