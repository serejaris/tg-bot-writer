Этап 2 — в работе

Как «Рецепт дня» сам будет выходить в Telegram

Простой план без программирования с вашей стороны. Вы по-прежнему придумываете меню и тексты — а канал будет публиковать их по расписанию, как будильник для постов.

✓ Уже сделано (этап 1)

  • 30 рецептов на завтрак и обед
  • Меню и контент-план на 30 дней
  • Списки покупок по неделям
  • Правила для агента, который пишет тексты

Всё это лежит в папке проекта — тексты готовы, осталось «подключить провод» к Telegram.

Шаги — по порядку

Каждый шаг — отдельная задача на GitHub. В каждой — кто что делает: 👤 вы или 🤖 агент в Cursor.

1

Создать канал и описать его

Завести Telegram-канал «Рецепт дня», придумать @username, записать ссылку в профиль канала проекта.

👤 Вы
  • Создать канал в Telegram (или выбрать существующий)
  • Придумать @username и прислать агенту публичную ссылку
  • После шага 2 — добавить бота администратором с правом публиковать посты
🤖 Агент
  • Записать название, @username и ссылку в channel-brief.md
  • Дать пошаговую подсказку, как добавить бота админом (когда бот будет готов)
  • Обновить задачу #3 на GitHub по мере готовности
Задача #3 на GitHub →
2

Завести бота-помощника

Через BotFather создаётся «робот», который умеет отправлять сообщения в ваш канал. Пароль бота хранится отдельно — не в интернете для всех.

👤 Вы
  • Написать @BotFather и создать бота
  • Скопировать токен и передать агенту безопасно (не в чат, не в GitHub) — например, в локальный файл .env
  • Добавить бота админом канала (шаг 1)
🤖 Агент
  • Настроить .env и убедиться, что секреты не попадают в git
  • Написать простую команду «отправить тестовый пост в канал»
  • Объяснить, что делать, если токен или права неверные
Задача #4 на GitHub →

После шага 1 можно параллельно: шаг 2 (бот) и шаг 3 (оформление) ↓

3

Красиво оформить пост в Telegram

Текст из файла должен правильно выглядеть в канале: жирный заголовок, список продуктов, совет в «спойлере».

👤 Вы
  • Открыть тестовый пост в канале и сказать: «нравится» или «поправить»
  • Уточнить, если что-то обрезалось или выглядит криво
🤖 Агент
  • Написать преобразование текста из файла в формат Telegram
  • Проверить на реальных постах из папки posts/scheduled/
  • Учесть правила из channel-brief.md: жирный, спойлер, моноширинные ингредиенты
Задача #5 на GitHub →
4

Настроить расписание

Каждый день в 8:00 — пост с завтраком и обедом. Каждое воскресенье в 10:00 — список покупок на неделю. Бот сам находит нужный файл по дате в контент-плане.

👤 Вы
  • Решить, где будет «будильник» (ваш Mac, сервер, облако) — агент подскажет варианты
  • Один раз проверить: пост вышел в 8:00 или 10:00
  • Следить, что в контент-плане на дату есть готовый файл
🤖 Агент
  • Настроить автозапуск по расписанию (время Аргентины)
  • Связать дату → строка в content-plan.md → файл поста
  • Написать инструкцию «как включить / выключить» простым языком
Задача #6 на GitHub →
5

Архив опубликованного

После выхода поста система помечает его «опубликован» и убирает в архив — чтобы не публиковать дважды и видеть, что уже вышло.

👤 Вы
  • Почти ничего — всё автоматически
  • По желанию: заглянуть в контент-план и убедиться, что статус «опубликован»
🤖 Агент
  • После отправки в канал — обновить статус в content-plan.md
  • Перенести файл из «запланировано» в «архив» (posts/published/)
  • Защита от двойной публикации одного и того же поста

  • Кратко описать: как вручную отправить пост и как работает расписание
  • Согласовать с правилами в AGENTS.md
