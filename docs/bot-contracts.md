# Контракты учебного Telegram-бота

Этот документ фиксирует поведение бота до написания кода. По нему пишутся тесты и проверяется реализация.

## Переменные окружения

| Переменная | Обязательна | Значение |
|---|---:|---|
| `TELEGRAM_BOT_TOKEN` | да | токен Telegram-бота |
| `TELEGRAM_CHANNEL_ID` | да | ID или username канала |
| `GLM_API_KEY` | да | ключ GLM API |
| `GLM_MODEL` | да | модель GLM, например `glm-5.2` |
| `GLM_BASE_URL` | нет | адрес API, если нужен не дефолтный |

Секреты читаются только из окружения или локального `.env`. В коде и тестах реальных ключей нет.

## Сценарий

1. Пользователь отправляет боту исходную идею.
2. Бот читает `channel-brief.md`.
3. Бот просит GLM предложить 3 направления.
4. Бот просит GLM предложить 3 хука.
5. Пользователь выбирает направление и хук.
6. Бот просит GLM написать 1 готовый пост.
7. Бот показывает пост пользователю.
8. Пользователь выбирает действие: сохранить в контент-план или опубликовать сразу.
9. При сохранении бот спрашивает дату публикации.
10. При публикации бот ждёт явное подтверждение.

## Данные внутри бота

### Идея пользователя

```json
{
  "text": "пост про быстрый завтрак из остатков риса",
  "chat_id": 123,
  "user_id": 456
}
```

`text` не должен быть пустым.

### Направления

```json
{
  "directions": [
    {"id": "d1", "title": "Практичный завтрак", "summary": "Как быстро использовать остатки"},
    {"id": "d2", "title": "Детский заход", "summary": "Как сделать блюдо понятным детям"},
    {"id": "d3", "title": "Экономия продуктов", "summary": "Как меньше выбрасывать"}
  ]
}
```

Всегда ровно 3 направления. У каждого направления есть `id`, `title`, `summary`.

### Хуки

```json
{
  "hooks": [
    {"id": "h1", "text": "Остался рис со вчера? Это уже завтрак."},
    {"id": "h2", "text": "Пять минут — и дети едят не бутерброды."},
    {"id": "h3", "text": "Самый спокойный завтрак начинается с холодильника."}
  ]
}
```

Всегда ровно 3 хука. Хук — первые 1-2 строки будущего поста.

### Выбор пользователя

```json
{
  "direction_id": "d1",
  "hook_id": "h2"
}
```

Бот принимает только `id` из показанных пользователю вариантов.

### Готовый пост

```json
{
  "title": "Быстрый завтрак из риса",
  "topic": "быстрый завтрак из остатков",
  "tags": ["завтрак", "остатки", "семья"],
  "body": "Текст поста для Telegram"
}
```

`body` должен быть готов к отправке в Telegram: без служебных комментариев агента и без заголовков `#`.

### Действие после поста

```json
{
  "action": "save",
  "date": "2026-06-26"
}
```

Допустимые действия:

- `save` — сохранить пост в репозиторий и контент-план;
- `publish_now` — опубликовать сразу после подтверждения.

Для `save` нужна дата публикации. Для `publish_now` нужно явное подтверждение пользователя.

## Markdown-файл поста

Файл сохраняется в `posts/drafts/` или `posts/scheduled/`.

```yaml
---
title: "Быстрый завтрак из риса"
date: 2026-06-26
status: scheduled
topic: "быстрый завтрак из остатков"
tags: [завтрак, остатки, семья]
source: "telegram-bot"
created: 2026-06-25
---
```

После frontmatter идёт только текст поста.

## Контент-план

При сохранении бот обновляет только `content-plan/content-plan.md`.

Минимально нужно отразить:

- дату публикации;
- статус `draft` или `scheduled`;
- ссылку на файл поста;
- короткую заметку по теме.

## Публикация

Бот публикует в Telegram-канал только после явного подтверждения пользователя.

После успешной публикации:

- пользователь получает итоговое сообщение;
- пост получает статус `published`;
- файл переносится в `posts/published/`, если такой перенос реализован на этом шаге.

## Ошибки

Ошибки показываются человеку простым текстом.

| Ситуация | Ответ бота |
|---|---|
| Пустая идея | `Пришли идею поста одним сообщением.` |
| Нет ключа GLM | `Не настроен ключ GLM. Проверь переменные окружения.` |
| GLM не ответил | `Не получилось получить текст от GLM. Попробуй ещё раз.` |
| Нет Telegram-канала | `Не настроен канал для публикации.` |
| Публикация не прошла | `Не получилось опубликовать пост. Проверь права бота в канале.` |
| Дата не распознана | `Напиши дату публикации в формате YYYY-MM-DD.` |

## Что тестировать

- пустая идея не запускает генерацию;
- по идее создаются 3 направления;
- по идее создаются 3 хука;
- после выбора создаётся 1 готовый пост;
- `save` требует дату;
- `publish_now` требует подтверждение;
- markdown-файл содержит валидный frontmatter;
- контент-план обновляется только в `content-plan/content-plan.md`;
- публикация вызывает Telegram API с готовым текстом;
- секреты читаются из окружения.
